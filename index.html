Project Overview

This project implements and compares two algorithms for finding the densest subgraph in a network, demonstrating significant performance improvements with the core-based approach.

Project Description

In this project, I've implemented two algorithms from the paper "Efficient Algorithms for Densest Subgraph Discovery" to find the densest subgraphs in various network datasets. The algorithms are:

  • Algorithm 1 (Exact): The traditional approach using flow networks
  • Algorithm 4 (CoreExact): An optimized approach using k-core decomposition

The densest subgraph discovery problem is fundamental to graph mining and has applications in network science, biological analysis, and graph databases. A densest subgraph is a subgraph with maximum edge density (the ratio of edges to vertices).

Algorithm Implementation

The project implements two key algorithms from the paper for finding the densest subgraph in a network.

1

Algorithm 1: Exact

Traditional exact algorithm using flow networks

Algorithm 1 is the traditional approach to finding the densest subgraph. It uses a parametric maximum flow algorithm with binary search to identify the subgraph with the highest density.

Key Implementation Details

  • Uses maximum flow computation on a specially constructed flow network
  • Performs binary search on density parameter α
  • Builds a flow network on the entire graph in each iteration
  • Has a time complexity of approximately O((mn + m³) log n) where n is the number of vertices and m is the number of edges

The algorithm works by iteratively guessing a density value and checking if there's a subgraph with at least that density. In each iteration, it constructs a flow network where:

  • Source node is connected to all vertices with capacity m
  • All vertices are connected to sink with capacity m+2α-deg(v)
  • Each edge (u,v) has two directed edges with capacity 1

Through binary search, the algorithm converges on the densest subgraph.

4

Algorithm 4: CoreExact

Optimized exact algorithm using k-core decomposition

Algorithm 4 (CoreExact) represents a significant advancement in densest subgraph discovery. It leverages k-core decomposition to dramatically reduce computation time while preserving result accuracy.

Key Implementation Details

  • Uses k-core decomposition to locate the densest subgraph in smaller subgraphs
  • Employs three key optimizations:
    • Tighter bounds on α: Uses properties of k-cores to establish better bounds
    • Locating CDS in specific cores: Focuses search on relevant k-cores
    • Progressively smaller flow networks: Reduces network size in each iteration
  • Still produces exact results just like Algorithm 1

The algorithm first performs core decomposition and locates the densest subgraph in a specific k-core, which is often much smaller than the entire graph. It then builds a flow network on this core and uses binary search similarly to Algorithm 1.

As the lower bound of the density increases during the binary search, CoreExact can locate the densest subgraph in a core with an even higher core number, leading to increasingly smaller flow networks and faster convergence.

Implementation Language and Environment

Both algorithms were implemented in C++ to maximize performance. The implementation includes:

  • A common graph data structure with optimizations for efficiently representing and manipulating graphs
  • A flow network implementation using Dinic's algorithm for maximum flow computation
  • K-core decomposition algorithms with bin-sort optimization
  • Comprehensive timing and result verification mechanisms

All experiments were conducted on a machine with an Intel(R) Xeon(R) 3.40GHz processor, 16 cores, and 125GB of memory, running Ubuntu.

Experimental Results

The experiments demonstrate that CoreExact significantly outperforms the traditional Exact algorithm while maintaining solution accuracy.

Execution Time Comparison

Performance Analysis

    • Dramatic Speedup: Algorithm 4 (CoreExact) consistently achieves significantly faster execution times compared to Algorithm 1 (Exact) across all datasets.
    • Massive Gap on AS-Caida: On the AS-Caida dataset, Algorithm 1 takes about 1,768,273 ms while Algorithm 4 finishes in just 526 ms, demonstrating an extraordinary speedup.
    • Superior Performance on All Datasets: Even on smaller datasets like Yeast, NetScience, and AS-733, Algorithm 4 drastically reduces computation time, showing its overall robustness and efficiency.
    • Scalability Advantage: Algorithm 4 handles both small and large graphs efficiently, highlighting its better scalability compared to Algorithm 1.

Key Findings

  1. Significant Performance Improvement: CoreExact achieves speedups ranging from 59× to 3,362× compared to the traditional Exact algorithm.
  2. Scaling with Graph Size: The performance advantage of CoreExact becomes more pronounced as the graph size increases.
  3. Perfect Solution Quality: Both algorithms found identical densest subgraphs across all datasets, confirming that CoreExact doesn't compromise on solution quality.
  4. Core Decomposition Efficiency: The overhead of performing core decomposition is negligible compared to the savings it generates in flow computation.
  5. Flow Network Reduction: CoreExact dramatically reduces flow network sizes through k-core localization, with reductions up to 95% in some cases.

Conclusion and Future Work

The core-based approach significantly advances the state-of-the-art in densest subgraph discovery.

Key Achievements

This project successfully implemented and compared two algorithms for densest subgraph discovery:

  • Successfully implemented Algorithm 1 (Exact) and Algorithm 4 (CoreExact) from the paper "Efficient Algorithms for Densest Subgraph Discovery"
  • Demonstrated dramatic performance improvements through the core-based approach
  • Verified the accuracy of results across multiple real-world datasets
  • Provided empirical evidence that confirms the theoretical advantages of the core-based approach

Future Directions

Several avenues for future research and development include:

  • Extending the algorithms to weighted graphs and directed networks
  • Implementing parallellized versions to further improve performance

  • Combining with approximation techniques for ultra-large-scale graphs
